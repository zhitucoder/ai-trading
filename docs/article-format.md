# 财报分析文章格式规范

> 本文档记录了财报分析文章的写作与排版要求，后续所有文章按此规范执行。

---

## 一、写作规范

### 1.1 人称与署名
- **"浩哥"**
- 第一人称用"我"（浩哥）

### 1.2 分析框架命名
- "张新民教授'战略视角财务报表分析'框架" → **"AI蒸馏专家'战略视角财务报表分析'框架"**
- 文中所有"张新民"三字替换为**"AI蒸馏专家"**
- 框架名"战略视角财务报表分析"保留不动

### 1.3 文章开头
- 封面图后，正文前，固定一段自我介绍：
  > **我是浩哥，用AI帮你读懂公司财报。**
- 可根据文章主题微调后半句，例如："用AI帮你分析股票"、"用AI帮你排雷"等
- 单独成段，不加粗以外的修饰

### 1.4 文章结构
1. **封面区**：标题 + 核心指标标签 + "浩哥 AI 分析"
2. **自我介绍**：我是浩哥，用AI帮你读懂公司财报
3. **基本画像**：公司业务定位 + 战略类型判断
3. **资产负债表重构**：经营/投资资产 + 经营/金融负债分类
4. **造血能力分析**：合同负债/核心指标
5. **核心利润拆解**：核心利润计算 + 利润结构分解
6. **三个匹配测试**：固定资产vs营收 / 营收vs核心利润 / 核心利润vsOCF
7. **关键风险信号**：应收账款/存货/商誉等
8. **综合诊断**：面子·日子·底子 + 最终结论
9. **文末注明**：系统生成 + GitHub地址 + 互动引导

### 1.5 指标计算公式（绕过损坏字段）
| 指标 | 计算方式 |
|------|---------|
| 核心利润率 | 核心利润 / 营业收入 × 100% |
| 净利润率 | 净利润 / 营业收入 × 100% |
| 合同负债/总资产 | fin_balance_sheet 直接取值 |
| ROE | fin_income.净利润 / fin_balance_sheet.净资产 × 100% |
| 资产负债率 | fin_balance_sheet.总负债 / fin_balance_sheet.总资产 × 100% |

## 二、排版规范

### 2.1 封面图
- 背景色：**暗黑调** `#1a1a2e`（不要蓝色渐变）
- 尺寸：860×540px（公众号封面比例）
- 文字：**居中对齐**
- 标题：白色 `#ffffff`，34px，加粗
- 高亮数字：金色 `#ffd700`
- 标签："浩哥 AI 分析"（灰底细边框）
- 副信息："2025年报 · 2026Q1" 等

### 2.2 配图处理
- **所有图片不走 imgbb 图床**
- 统一使用 **base64 data URI** 直接嵌入 markdown / HTML
- 格式：`![](data:image/png;base64,...)`

### 2.3 表格 → 截图
- 所有 markdown 表格和代码块中的结构化数据，**全部转为截图**（不留文本表格）
- 截图风格（暗调）：
  - 背景：`#1a1a2e`
  - 表头行：`#16213e` 背景，白色 `#ffffff` 加粗字体
  - 分隔线：`#0f3460`
  - 第一列（标签列）：亮色（`#8bbaff` 浅蓝 或 `#ffd700` 金色），加粗
  - 数值列：浅灰 `#e0e0e0`，重要值用金色或红色高亮
  - 说明文字：`#666` 或 `#888`
  - 奇数行：`#1a1a2e`，偶数行：`#1e1e36`
   - 底部注：灰色小字 `数据来源：ai_trading 财务数据库`
- **截图底部必须空出2行空白**（用于叠加水印）：在 `</tbody>` 前加2行 `<tr><td style="padding:4px 8px" colspan="10">&nbsp;</td></tr>`


### 2.4 代码块中的结构化数据
- 资产结构（经营/投资分类）、负债结构、利润拆解等
- 统一转换为暗调截图，规则同 2.3

### 2.5 生成公众号预览
- 工具：`wechat-copy.js <input.md> --preset "墨色书香"`
- 生成命令：
  ```
  cd analysis/
  node /home/rick/.claude/skills/wechat-article-typeset/wechat-copy.js "文件名.md" --preset "墨色书香"
  ```
- 输出：`article.preset.html` + 预览链接
- 图片全部 base64 嵌入后运行

## 三、文件管理

### 3.1 封面图片
- 个股分析：保存到 `analysis/images/{股票代码}-cover.png`
  - 例如：`analysis/images/cssb-cover.png`
- 专题文章：保存到 `analysis/images/{主题简称}-cover.png`
  - 例如：`analysis/images/fee-cover.png`（财务费用专题）

### 3.2 分析文章
- 保存到 `analysis/` 目录
- 文件名：`{公司名}_{框架名}.md`
- 例如：`中国船舶_张新民视角分析.md`
