## 12.2 AI 编程核心心法

**心法一：描述需求比写代码更重要**

```
差的描述 → AI 理解模糊 → 结果不对 → 反复改
好的描述 → AI 一次到位 → 验证通过 → 继续下一步
```

**怎么描述好需求：**
1. **说清楚输入输出** — "输入股票代码，显示最近 30 天的收盘价"
2. **举例子** — "就像同花顺的 F10 页面那样"
3. **给约束** — "数据从数据库读，不要硬编码"
4. **一次一件事** — 不要一次给太多需求

**心法二：迭代式开发**

```
第一轮：告诉 Claude「做这个功能」
第二轮：检查结果，告诉 Claude「这里不对，改成 xxx」
第三轮：继续调整，直到满意
```

**AI 编程不是一次到位的，是逐步迭代到满意的。** 每次只改一个问题，不要攒一堆问题一次性说。

**心法三：让 AI 帮你验证**

```
做完了 → 让 AI 检查 → "帮我看看这段代码有没有 bug"
改完了 → 让 AI 审查 → "检查我改的有没有问题"
有 bug → 把错误信息告诉 AI → "报错了，帮我修"
```

## 12.3 自学路径

**路径一：读一本交易书籍，让 AI 帮你实现**

买一本交易经典，比如：
- 《股票魔法师》— Mark Minervini
- 《股票作手回忆录》— 利弗莫尔
- 《聪明的投资者》— 本杰明·格雷厄姆

**做法：**
1. 读完一个策略
2. 告诉 Claude"帮我实现这个选股条件"
3. 用本课程的系统做回测验证
4. 看策略在 A 股历史上能不能赚钱

**路径二：蒸馏更多名人**

不只是交易大师，你可以蒸馏任何人：
- 企业家的思维方式（马斯克、任正非）
- 哲学家的思维模型（亚里士多德、尼采）
- 行业专家的分析框架（不限于投资）

**路径三：独立项目**

选一支你关注的股票，完成完整闭环：
```
数据 → 策略筛选 → 回测验证 → AI 辩论 → 分析参考
```

## 12.5 Q&A + 毕业项目选题

**常见问题：**

**Q：我完全不懂编程，能做这门课吗？**
A：这门课就是为你设计的。你只需要会描述需求，代码的事交给 AI。

**Q：没有通达信数据怎么办？**
A：可以告诉 Claude 去网上找免费的股票数据源替代。

**Q：Mac 电脑怎么装？**
A：Mac 自带 Linux 环境，不用装 WSL，其他步骤一样。

**Q：学完这门课能做什么？**
A：你掌握了 AI 编程的方法论，能自己搭建量化分析工具，把任意人物蒸馏为 AI 角色辅助分析。但量化工具是辅助决策的，不保证任何收益。股市有风险，投资需谨慎。

**毕业项目想法：**
1. 蒸馏你最喜欢的投资大师，问他对当前市场的看法
2. 回测你一直想试的"低买高卖"策略，看历史数据上能否赚钱
3. 做一个选股组合，每天自动筛选符合条件的股票
4. 把你的分析结果做成报告页面分享给朋友

## 动手环节

每人说出一个自己的交易想法，现场让 Claude 验证可行性。

**示例：** "我听说 MACD 金叉买入很准，帮我回测一下 A 股过去 5 年所有 MACD 金叉信号的收益率。"
