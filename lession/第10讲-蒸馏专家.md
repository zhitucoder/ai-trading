## 10.1 蒸馏名人概念与意义

**什么是蒸馏？**

把一个人的思维方式、决策逻辑、表达风格提炼成一套提示词，让 AI 学会用这个人的视角看问题。

**蒸馏哪些内容？**

| 维度 | 内容 | 来源 |
|------|------|------|
| 思维框架 | 核心心智模型、分析框架、判断标准 | 著作、演讲、访谈 |
| 决策逻辑 | 怎么做决策、依赖什么信号、什么情况下行动 | 经典案例、实战记录 |
| 表达方式 | 语言风格、常用类比、叙事结构 | 公开言论、文字作品 |
| 时间线 | 不同阶段的认知演变、关键转折点 | 传记、媒体报道 |
| 他者评价 | 别人怎么评价他、他的影响力来源 | 同行评价、学界研究 |

**为什么要蒸馏名人？**

```
读书只能读到知识 → 蒸馏能让 AI 用他的思维框架分析你的问题
巴菲特写了 60 年股东信 → 蒸馏后你能直接问他「这支股票巴菲特怎么看」
利弗莫尔写了《股票作手回忆录》→ 蒸馏后 AI 能用他的关键点交易法分析行情
张新民教了 30 年财报分析 → 蒸馏后 AI 能像他一样扒公司底裤
```

**蒸馏的本质：** 思维框架的显式化。不是让 AI 模仿说话，而是让 AI 理解这个人怎么思考、怎么决策。

## 10.3 女娲造人 Skill — 自动蒸馏任意人物

**女娲（huashu-nuwa）** 是一个自动蒸馏技能：输入人名或主题，它自动完成深度调研 → 思维框架提炼 → 生成可运行的 Skill 文件。

**GitHub 地址：** https://github.com/alchaincyf/nuwa-skill

**安装方法：**

告诉 Claude：

> "帮我安装女娲造人技能：https://github.com/alchaincyf/nuwa-skill"

> ⚠️ 也可以用终端命令 `npx skills add alchaincyf/nuwa-skill`，但在有多个 Claude/OpenCode 环境时安装目录不确定。建议直接告诉 AI 安装。

安装后即位于 `~/.claude/skills/huashu-nuwa/`，刷新即可在蒸馏专家列表中使用。

**工作原理：**

```
输入人名（如"巴菲特"）
  → 自动搜索著作、演讲、访谈等公开资料
  → 从 6 个维度提炼（思维框架、决策逻辑、表达方式、时间线、他者评价、关键案例）
  → 生成标准化的 SKILL.md 文件
  → 安装到 ~/.claude/skills/ 目录
  → 刷新页面即可使用
```

**本地素材要求（可选）：**

如果你有自己的独家素材（书籍、访谈等），可以提供给 Claude 作为蒸馏素材。素材放入技能目录下的对应子目录：

```
~/.claude/skills/[人物名]-perspective/
└── references/
    └── sources/
        ├── books/         # 书籍 PDF / epub
        ├── transcripts/   # 演讲/访谈文本、视频字幕 SRT
        └── articles/      # 博客、文章、内部文档
```

| 素材类型 | 放入目录 | 说明 |
|---------|---------|------|
| 书籍 PDF | `sources/books/` | 完整电子书 |
| 演讲/访谈文本 | `sources/transcripts/` | 纯文本或 Markdown |
| 视频字幕 SRT | `sources/transcripts/` | 去时间戳后阅读 |
| 博客/文章 | `sources/articles/` | Markdown 或网页链接 |
| 内部文档/备忘录 | `sources/` | 直接放入即可 |

素材越多、越一手（原书 vs 二手转述），蒸馏出来的质量越高。

## 10.6 蒸馏专家做产品经理

让投资大师帮你设计产品功能。这里展示两种指令方式，体会差异。

**专家版指令**（详细告诉 AI 要做什么）：

> "切换到股票魔法师 Mark Minervini 的视角：
> 
> 我用 Minervini 的 VCP（波动收缩）选股方法，帮我设计一个 VCP 选股页面：
> - 全市场扫描正在 VCP 收缩的股票
> - 显示收缩次数（T1、T2、T3...）
> - 显示收缩幅度
> - 标注突破信号
> 
> 实现这个页面，把我的选股逻辑变成可视化的工具。"

**蒸馏版指令**（不告知细节，让它自己设计）：

> "切换到 Minervini 视角，用他的选股方法，帮我设计一个选股页面。"

**对比两个指令的差异：**

专家版指令需要你自己懂 VCP 的细节，把每一步都列出来——扫描什么、显示什么、标注什么，相当于你在教 AI 怎么做。蒸馏版指令只有一句话，你没有提任何细节，但 Minervini 的 SEPA 交易系统和 VCP 选股方法已经在他的 Skill 里了，他会自动按自己的框架来设计。结果可能差不多，但蒸馏版指令背后释放了一个信号：你不需要懂他的方法细节，你借的是他的大脑。这就是蒸馏的价值——把方法论装进 Skill，指令就可以从「手把手教」退化到「一句话触发」。

**Claude 会做什么：**
- 加载 Minervini 的思维框架
- 设计 VCP 检测算法
- 生成 VCP 选股页面
- 显示筛选结果和分析

## 10.7 蒸馏专家网页服务

将蒸馏专家做成网页服务，用户选择专家、输入股票代码、提出自己的问题，AI 以该专家的视角分析真实数据并回答。

### 架构

```
浏览器                     FastAPI                         MySQL
  │                          │                              │
  │  POST /api/expert/ask    │                              │
  │  {expert_id, stock_code, │                              │
  │   question}              │                              │
  │ ──────────────────────►  │  查 K线/财务/画像            │
  │                          │ ──────────────────────────►  │
  │                          │ ◄──────────────────────────  │
  │                          │                              │
  │                          │  加载 Expert SKILL.md        │
  │                          │  └→ 构造 System Prompt       │
  │                          │                              │
  │                          │  (AI 以专家视角回答)          │
  │ ◄──────────────────────  │                              │
  │ {answer}                 │                              │
```

### 后端 API

新增路由 `src/app/routers/expert.py`，提供三个接口：

| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/expert/list` | GET | 自动扫描 `~/.claude/skills/` 下已安装的蒸馏技能 |
| `/api/expert/profile/{expert_id}` | GET | 加载指定技能的思维框架（读取 SKILL.md） |
| `/api/expert/ask` | POST | 用户提问 → 注入真实数据 + 思维框架 → AI 回答 |

**三个步骤实现：**

**第一步：自动发现技能**

后端扫描 `~/.claude/skills/` 目录，识别蒸馏技能（命名规则 `*perspective/SKILL.md`）：

```python
import os
from pathlib import Path

SKILLS_DIR = Path.home() / '.claude' / 'skills'

def discover_experts():
    experts = []
    for d in SKILLS_DIR.iterdir():
        if not d.is_dir():
            continue
        skill_file = d / 'SKILL.md'
        if not skill_file.exists():
            continue
        content = skill_file.read_text(encoding='utf-8')
        name = _extract_name(d.name, content)
        experts.append({
            'id': d.name,
            'name': name,
            'description': _extract_description(content),
        })
    return experts
```

**第二步：构造思维框架 Prompt**

将 SKILL.md 中提炼的心智模型、决策启发式、表达DNA 等作为 System Prompt：

```
你正在扮演 {expert_name}。
以下是他的思维框架和表达方式，你必须严格遵循。

{skill_content}

用户的请求包含以下真实数据，请基于这些数据回答：
- K线数据: 最近200个交易日
- 财务数据: 近4个季度营收/利润/负债
- 股票画像: 阶段/评分/技术指标/业务标签
```

**第三步：注入真实数据**

在 `/api/expert/ask` 中，从数据库查询该股票的实时数据，组装到 Prompt 中：

```python
@app.post('/api/expert/ask')
def expert_ask(body: ExpertAskRequest):
    # 1. 扫描技能目录，加载选中专家的 SKILL.md
    skill = load_skill(body.expert_id)

    # 2. 查询股票数据
    klines = get_klines(body.stock_code, 200)
    financials = get_financials(body.stock_code)
    profile = get_profile(body.stock_code)

    # 3. 构造 system prompt
    system_prompt = f"""
    你正在扮演 {skill['name']}。
    以下是他的思维框架和表达方式：
    {skill['framework']}

    回答问题时必须：
    1. 以该人物的视角和风格分析
    2. 所有数字引用真实数据
    3. 末尾标注 "—以上为教学演示，不构成投资建议—
    """

    # 4. 调用 AI API 获取回答
    answer = call_ai_api(
        system=system_prompt,
        user=f"""
        股票: {body.stock_code} ({body.stock_name})
        用户问题: {body.question}

        实时数据:
        - 最新价: {profile['latest_price']}
        - 均线: MA5={ma5}, MA10={ma10}, MA20={ma20}
        - RSI14: {rsi14}
        - 营收增长率: {rev_growth}
        - 净利润增长率: {profit_growth}
        - 资产负债率: {debt_ratio}
        """
    )

    return {'answer': answer}
```

### 前端页面

页面结构：顶部专家下拉框 → 搜索栏（股票代码 + 文本输入）→ 对话区（专家头像 + 气泡回答）。

对话区支持 Markdown 渲染，每条回答底部显示"—以上为教学演示，不构成投资建议—"。

**关键交互逻辑：**
- 切换专家时清空对话历史
- 发送后自动滚动到最新回答
- 加载时显示打字机动画

### 自动发现机制

新安装的蒸馏技能无需任何配置即可在页面下拉框中出现。后端每次调用 `/api/expert/list` 时动态扫描 `~/.claude/skills/` 目录，前端在 `onMounted` 时加载列表。

### 约束

| 约束 | 说明 |
|------|------|
| 角色回答必须引用真实数据 | 所有数字来自 MySQL，不允许虚构 |
| 技能自动发现 | 新安装的技能自动出现在页面下拉框，零配置 |
| 末尾标注免责 | 每条 AI 回答底部必须标注"以上为教学演示，不构成投资建议" |
