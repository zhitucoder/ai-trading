# 股票画像预计算与筛选系统设计

## 1. 业务目标

- 全市场 A 股（约 5,500 只）画像预计算，每天只保留最近两天的数据
- 用户通过画像标签、阶段、评分等维度筛选股票
- 手动触发刷新，显示最新数据日期
- "股票画像" 页面两个 Tab：**单股画像**（现有）+ **画像筛选**（新增）

## 2. 架构总览

```
(手动触发)
 刷新按钮 ──→ batch_profile.py ──→ stock_profiles 表
                 ↑                        │
           并发池 (16 workers)             │
                                           ▼
  用户浏览器 ──→ /api/profiles/search ──→ SQL 查询 ──→ 筛选结果
                   ↑
  GET /api/profile/{code} ←──── stock_profiles 表 (profile_json)
```

## 3. 存储设计

## 4. 批量计算设计

### 4.1 batch_profile.py

独立脚本，全量计算所有股票画像并写入 stock_profiles 表。写入完成后，删除 2 天前的旧数据。

```
流程:
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 1. 获取股票列表 │ → │ 2. 并发计算画像 │ → │ 3. 批量写入   │ → │ 4. 清理旧数据 │
│    (stocks表)  │    │  (16进程池)   │    │  (批处理INSERT)│    │  (删除2天前)  │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

```python
# batch_profile.py (伪代码)
from concurrent.futures import ProcessPoolExecutor, as_completed
from src.app.strategies.profile import generate_profile

WORKERS = 16
BATCH_SIZE = 500  # SQL 批量写入每批数量

def compute_batch():
    stocks = get_all_stock_codes()
    total = len(stocks)
    computed = 0
    errors = 0
    batch_rows = []

    with ProcessPoolExecutor(max_workers=WORKERS) as pool:
        futures = {pool.submit(generate_profile, code): code for code in stocks}
        for future in as_completed(futures):
            result = future.result()
            computed += 1
            if 'error' not in result:
                batch_rows.append(_to_profile_row(result))
            else:
                errors += 1
            # 每 BATCH_SIZE 条写入一次
            if len(batch_rows) >= BATCH_SIZE:
                _batch_insert(batch_rows)
                batch_rows.clear()
            # 更新进度到 refresh_log
            _update_progress(computed, total, errors)

    # 写入剩余数据
    if batch_rows:
        _batch_insert(batch_rows)

    # 清理 2 天前的数据
    _clean_old_data()

def _clean_old_data():
    """只保留最近 2 天的数据"""
    delete_old = "DELETE FROM stock_profiles WHERE trade_date < DATE_SUB(CURDATE(), INTERVAL 2 DAY)"
    execute(delete_old)
```

**性能估算：**
- 单只画像 ≈ 300ms（含 4 次 SQL 查询）
- 16 进程全量 ≈ 3-5 分钟
- 手动触发后，用户可通过进度轮询看到完成百分比

## 5. API 设计

### 5.1 画像筛选 API

#### GET /api/profiles/status

返回当前预计算状态。

```json
{
  "latest_data_date": "2026-06-30",
  "latest_trade_date": "2026-06-30",
  "fin_report_date": "2026-03-31",
  "total_profiles": 5512,
  "last_refresh_time": "2026-06-30 18:00:00",
  "refreshing": false
}
```

#### POST /api/profiles/refresh

触发全量刷新。检查是否已在刷新中，防止重复触发。

```json
// Response 202
{
  "status": "started",
  "total_stocks": 5512,
  "estimated_minutes": 5
}
```

#### GET /api/profiles/refresh/progress

轮询刷新进度。

```json
{
  "status": "running",
  "total": 5512,
  "computed": 3200,
  "errors": 2,
  "elapsed_seconds": 120,
  "estimated_remaining_seconds": 90
}
```

#### POST /api/profiles/search

**SQL 生成逻辑：**

```sql
SELECT stock_code, stock_name, latest_price, price_change_pct,
       stage_id, stage_confidence, tech_score, fund_score,
       revenue_growth, net_profit_growth, debt_ratio,
       profile_json
FROM stock_profiles
WHERE trade_date = (SELECT MAX(trade_date) FROM stock_profiles)
  AND stage_id IN ('stage.s2', 'stage.s1s2')
  AND tag_bullish = TRUE
  AND tag_high_growth = TRUE
  AND tag_financial_healthy = TRUE
  AND tag_bearish = FALSE
  AND tag_profit_collapse = FALSE
  AND (tag_volume_surge = TRUE OR tag_volume_price_up = TRUE)
  AND tech_score >= 60
  AND fund_score >= 50
  AND revenue_growth >= 20
  AND (debt_ratio IS NULL OR debt_ratio < 60)
ORDER BY tech_score DESC
LIMIT 50 OFFSET 0;
```

### 6.2 交互设计

- **Tab 切换**：`v-if` 切换单股画像模板 vs 画像筛选模板
- **刷新按钮**：放在 Topbar 右侧，刷新中变为旋转加载 + 进度百分比
- **最新数据日期**：从 `GET /api/profiles/status` 获取，显示在 Topbar
- **筛选条件**：Vue 响应式表单，点击「执行筛选」触发 `POST /api/profiles/search`
- **标签选择**：分组展示复选框组（均线/RSI/成交量/成长性/衰退/财务健康/趋势动量/连续增长）
- **结果点击**：点击某行跳转到单股画像 Tab 并填入股票代码
- **单股画像回退**：若有缓存（`stock_profiles.profile_json`）则直接渲染；否则调用实时 API
