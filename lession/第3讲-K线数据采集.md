## 3.1 系统整体架构回顾

在进入数据采集之前，先回顾一下整个系统的五层架构：

<img src="minervini-architecture.svg" alt="星火量化交易系统架构" width="100%">

| 层 | 技术 | 功能模块 |
|---|---|---|
| **Web UI** | Vue 3 SPA | VCP 选股 · 股票画像(61标签) · 策略回测 · 蒸馏专家 · 数据管理 |
| **FastAPI 应用层** | 路由分发 + Pydantic | vcp/api · tags/api · backtest · ai-ask(SSE) · data_mgmt |
| **业务逻辑层** | 核心算法 | VCP Engine · Tags Calculators · Backtest Engine · Debate Engine · Data Import |
| **数据访问层** | SQLAlchemy ORM | SessionLocal / Engine / DeclarativeBase |
| **MySQL 8.0** | localhost:3306 / ai_trading | 行情 · 财务 · 标签 · 回测 四大板块 |

**第 3 讲的任务**就是搞定最底层——把通达信的日K线数据解析后写入 MySQL 的 `daily_prices` 表，为上面所有业务逻辑提供数据基础。

数据流方向：**UI → API → 业务逻辑 → ORM → DB**，每一层通过箭头串联。

### 方式二：用 DBeaver 自己看

如果你装了 DBeaver（数据库图形化工具），可以直接连上去翻数据：

1. 打开 DBeaver，新建连接 → MySQL
2. 填以下信息：
   - **Host：** `localhost`
   - **Port：** `3306`
   - **Database：** `ai_trading`
   - **用户名：** `root`
   - **密码：** `aitrading123`
3. 连接后展开 `ai_trading` → `表`，你会看到：
   - `stocks` — 股票基本信息表
   - `daily_prices` — 日K线数据表（打开就能看到所有股票的历史行情）
4. 对着 `daily_prices` 点右键 → `查看表` → 随便浏览数据，或点 `SQL` 标签自己写 SQL 玩
